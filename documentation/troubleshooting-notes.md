# Troubleshooting Notes

This document provides basic troubleshooting notes for the simulated office and warehouse network.

These notes are written from an IT Support / Junior Network Support perspective and are intended for portfolio demonstration only.

## Common Issue: User Has No Network Connection

Suggested checks:

1. Confirm whether the issue affects one user or multiple users.
2. Check whether the user is connected by wired network or Wi-Fi.
3. Confirm the wall outlet or wireless access point location.
4. Check the patch panel map and switch port map.
5. Confirm whether the endpoint is in the expected network segment.
6. Test another known working cable or port if available.
7. Record the outcome and update documentation if the port label is incorrect.

## Common Issue: Printer Unavailable

Suggested checks:

1. Confirm whether other users can print.
2. Check device power and display panel status.
3. Confirm the printer is connected to the expected network segment.
4. Check the assigned switch port and patch panel record.
5. Confirm that the workstation uses the correct printer queue or driver.
6. Record any hardware, driver, or network change.

## Common Issue: Camera Offline

Suggested checks:

1. Confirm which camera is affected.
2. Check the camera device register for location, switch, and port.
3. Check whether the connected switch port is active.
4. Confirm that the recording system can see other cameras.
5. Check whether recent work occurred near the camera, patch panel, or rack.
6. Update the device register after any replacement or relocation.

## Common Issue: Guest Wi-Fi Not Working

Suggested checks:

1. Confirm whether the issue affects all guests or one device.
2. Check whether the access point is powered and reachable.
3. Confirm that corporate Wi-Fi is working separately.
4. Check the access point entry in the switch port map.
5. Record whether the issue is access point, internet, or client-device related.

## Documentation Follow-Up

After resolving an issue, update the relevant document if any information changed:

- Switch port map
- Patch panel map
- IP address plan
- Camera device register
- Network overview

Good documentation reduces repeat troubleshooting time and improves handover quality.
